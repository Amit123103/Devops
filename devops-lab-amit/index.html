🛠️ Experiment 1: Running Isolated Nginx Workloads

Aim: Setup, configure, and operate a containerized web service using Docker.

Prerequisites:

Ensure that Docker Desktop or a Linux host daemon is operational.

Execution Steps:
# Pulling official lightweight base
docker pull nginx:alpine

# Run containerized service in background mode
docker run -d --name amit_web_container -p 8080:80 nginx:alpine

# Print running container statuses
docker ps -a

Expected Output: The web container starts, binds to port 8080, and renders the default Nginx index page at http://localhost:8080.

🛠️ Experiment 2: Custom Node.js Microservice Generation

Aim: Build, tag, and persist customized local Node.js application images using a Dockerfile.

Execution Steps:
# Navigate to the context directory
cd docker/

# Compile the target Dockerfile image
docker build -t amit-node-app:v1 .

# Run the newly generated custom image
docker run -d --name node_test -p 3000:3000 amit-node-app:v1

Expected Output: Continuous console updates output build steps and successful tag confirmations.

Unit III: Microservices with Docker Compose

Coordinate multi-tier enterprise services seamlessly using Docker Compose.

Simulation Console
🛠️ Experiment 3: WordPress & MySQL Single-Script Deployment

Aim: Launching decoupled backend and web tier microservices using a unified script.

Execution Steps:
# Launch service stacks
docker compose -f compose/wordpress.yml up -d

# Verify operational running containers
docker compose -f compose/wordpress.yml ps

Expected Output: Dynamic isolation network created, database starts up, and WordPress binds to port 8080.
